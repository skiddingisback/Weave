Built for large production games

Hundreds of modules, dozens of services, multiple developers, CI, and hot reload.

◆

Typed dependency injection

Services are looked up by typed tokens with full autocomplete everywhere. Implementations never require each other, so big graphs stay cycle-free.

↯

Deterministic startup

Declared dependencies drive a topological boot order with cycle detection, fail-fast lifecycle phases, and reverse-order shutdown.

⌘

Schema-first networking

RPCs and events are validated on both sides against a shared schema, with rate limiting, middleware, and a version fingerprint to catch deploy skew.

◷

Diagnostics you can read

Startup timelines, dependency graphs exported to Mermaid & DOT, and structured errors with concrete fixes — not stack traces.

Quickstart

Define a typed contract, implement it, and bootstrap the app.

-- ShopService/definition.luau — the typed contract
local Weave = require(ReplicatedStorage.Weave)
export type API = { getBalance: (self: API, player: Player) -> number }
return { Token = Weave.token("ShopService") :: Weave.Token<API> }

-- ShopService/init.luau — the implementation
return Weave.service({
    token = Def.Token,
    dependencies = { DataDef.Token },
    create = function(ctx): Def.API
        local data = ctx:use(DataDef.Token) -- fully typed
        local self = {} :: Def.API
        function self.getBalance(_, player)
            return data:read(player, "balance")
        end
        ctx:onStart(function() ctx.log:info("shop open") end)
        return self
    end,
})

-- Server bootstrap
local app = Weave.App.new({ name = "Server" })
app:registerMany(require(ServerScriptService.Registry))
app:start()
app:bindToClose()
